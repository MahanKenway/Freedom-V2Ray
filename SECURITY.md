# Security Policy

## Scope

Freedom-V2Ray processes publicly available configuration data and publishes generated subscription files. The project is intended for lawful research, testing, automation, and network configuration management.

It is not intended for unauthorized access, credential theft, traffic interception, attacks against third-party systems, or abuse of third-party networks.

## Reporting a vulnerability

Please do not open a public issue for sensitive vulnerabilities.

Report security issues privately through GitHub Security Advisories or by contacting the maintainer. Include:

- affected file or component;
- reproduction steps;
- impact;
- suggested mitigation.

## Sensitive data

Do not submit:

- API keys;
- private keys;
- passwords;
- private server credentials;
- personal data;
- private configuration URLs.

## Handling third-party data

The maintainers do not operate or control third-party endpoints listed by external sources. Reported private or abusive data may be removed without prior notice.

## Supported versions

Only the latest version on the main branch is actively maintained.
